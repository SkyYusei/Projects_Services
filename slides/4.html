Microsoft Azure

Introduction to Azure

Microsoft Azure is a cloud computing platform and infrastructure, created by Microsoft, for building, deploying and managing applications and services through a global network of Microsoft-managed and Microsoft partner hosted datacenters. It provides both PaaS and IaaS services and supports many different programming languages, tools and frameworks, including both Microsoft-specific and third-party software and systems. You will work with setting up Azure Virtual Machines (VMs) and using its storage services for various projects in this course.

Video: Introduction to Azure (Source: Microsoft)

You can visit the Azure console either in Azure Portal or Azure Manage (Azure Portal is the newer console, so try to use it in this course when possible). For more information you can read through the Azure introduction here.

Microsoft Azure Compute

Introduction

Azure compute services provide virtual machines that you can configure and launch, log in, and install and run software. To launch an Azure virtual machine:

  1. Go to the Azure Portal and click on New -> Compute
  2. Select Ubuntu Server 14.04 LTS
  3. Under Select a Deployment Model, choose Resource Manager.
  4. You will need to specify a Name, User name and Password.
  5. You will also need to specify a Resource group, you can use or create a default resource group here.
  6. Leave the Location as East US by default.
  7. Next, you will be required to choose a size for your virtual machine. Click on View All and select the A0:Basic size.
  8. In the next section, you can leave most options as default. You can modify the Network security group to open up additional ports HTTP(80) and so on.
  9. Finally, in the Summary section, you can view all the options for this virtual machine. Click on OK to launch the virtual machine.

Deployment Models

Azure has two deployment models: Azure Service Manager (ASM, also known as Classic) and Azure Resource Manager (ARM). ARM is the newer management type and has improved APIs, hence in this course, we will use ARM wherever possible.

Note: If resources are deployed under one management type, they will not appear in the other. More information on ARMs can be found here.

Tasks to try:

You should get comfortable doing the following tasks:

  1. Configure and launch a VM with pricing tier of A0 standard using Ubuntu Server 14.04 LTS and the Resource Manager deployment model.
  2. Connect to the VM using SSH from your computer.
  3. SFTP/SCP a small file from your computer to the VM.
  4. Use any text editor on the instance to create a small script or program (such as a bash or python script) and run it on the VM instance.
  5. Terminate the VM and storage data disk when done.

Azure Storage

Azure storage is a unified storage service available to Azure users which provides highly scalable, durable and globally available storage for various application needs. Azure storage is offered in three forms:

  1. Azure Blobs, an object storage system, similar to Amazon's S3;
  2. Azure Tables, a database system;
  3. Azure Queues, a messaging queue for applications, and
  4. Azure Files, a file system.

We will look at Azure Blobs in this primer, and as you complete additional projects in this course, the differences between the various storage systems will be more apparent to you. To access the service, you must first create an Azure Storage account.

Creating an Azure Storage Account

To create an Azure storage account:

  1. Go to the Azure Portal and click on New -> Data + Storage.
  2. Select Storage account.
  3. You will need to specify a unique Storage account name.
  4. Under Pricing tier, click on View All and select the L:Locally Redundant pricing tier.
  5. Select the East US location.
  6. You can leave the Diagnostics unconfigured.

Your storage account will take a few minutes to get provisioned. Once provisioned, you have access to Blobs, Containers, Queues and File Shares. In the next section, we will work with Blob storage using the Azure CLI tool.

Azure CLI

Downloading and installing Azure CLI

Before this section, you should have already created your Azure account using instructions in the service set-up primer section. Go to Azure and at ‘Command-line Tools, download the respective client for the operating system you’re using. Follow through the instructions and complete the installation.

Figure: Downloading Azure CLI

Configuring Azure CLI

Do azure account download and a new window on your default browser should open up. Select the Microsoft account in which azure was setup in, then it will automatically download your subscription file.

Figure: Downloading Azure Account

Then import your account by running the command azure account import [file]. So now if you run the command azure account list you should be able to see your account along with your subscription information.

Figure: Importing Azure Account

By default azure mode is set to ASM or Azure Service Manager (v1), check by 

azure config list
. We want to use ARM or Azure Resource Manager (v2) by doing azure config mode arm to switch from ASM to ARM. The difference between the two modes can be found here.

Everything that you can do on manage.windowsazure.com can be done via the Azure CLI. For more information visit here.
